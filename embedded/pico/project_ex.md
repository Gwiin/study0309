**안 1. Wi-Fi 스마트 환경 모니터**

**1. 프로젝트 개요**
이 프로젝트는 3개의 Pico W가 각각 `센서 노드`, `제어 노드`, `모니터링 서버 노드` 역할을 맡아, 환경 데이터를 수집하고 네트워크로 공유하며, 특정 조건에서 액추에이터를 제어하는 분산형 IoT 시스템이다.

핵심 시나리오:
- 센서 노드가 온도, 조도, 가변저항 값 등을 읽어 서버 노드로 전송
- 서버 노드는 이를 저장, 표시, 판단
- 서버 노드는 조건에 따라 제어 노드에 LED/부저/팬 동작 명령 전달
- 제어 노드는 실제 장치를 동작시키고 결과를 서버에 응답

이 프로젝트는 `주기적 데이터 전송`, `상태 시각화`, `임계값 기반 제어`를 보여주기에 적합하다.

**2. 프로젝트 목표**
- Pico W의 Wi-Fi 소켓 통신 이해
- 센서 데이터 수집 및 주기적 네트워크 전송 구현
- 다중 MCU 협업 구조 설계
- 단순 로컬 제어가 아닌 분산 시스템 구현
- 임베디드 시스템에서 상태 판단과 원격 제어 경험

**3. 시스템 구성**
노드 구성:
- `노드 A`: 센서 노드
- `노드 B`: 제어 노드
- `노드 C`: 모니터링/서버 노드

구조:
```txt
[노드 A: 센서] ----Wi-Fi----\
                              > [노드 C: 서버/모니터] ----Wi-Fi---- [노드 B: 제어]
```

**4. 노드별 역할**
`노드 A: 센서 노드`
- 조도, 온도, 가변저항, 버튼 상태 읽기
- 일정 주기마다 센서 데이터 송신
- 버튼 입력 시 이벤트 메시지 전송
- 센서값 이상 여부를 로컬에서 1차 판단 가능

`노드 B: 제어 노드`
- 서버 명령 수신
- LED on/off 또는 밝기 조절
- 부저 동작
- 선택적으로 모터/팬 제어
- 동작 결과를 서버에 응답

`노드 C: 서버/모니터 노드`
- 다른 노드들의 소켓 연결 관리
- 센서 데이터 수신 및 최신 상태 저장
- 임계값 비교 후 제어 명령 생성
- 현재 상태를 OLED, UART 터미널, 간단한 웹 출력 중 하나로 표시
- 사용자 입력에 따라 제어 명령 전송

**5. 핵심 기능 명세**
기능 1. 센서 데이터 수집
- 노드 A는 1초 또는 2초 주기로 센서값 측정
- ADC 기반 아날로그 센서 사용 가능
- 측정값은 정수 또는 소수 한 자리까지 전송

기능 2. 상태 전송
- 노드 A는 측정값을 서버 노드에 주기적으로 전송
- 전송 실패 시 재시도 또는 다음 주기 재전송

기능 3. 임계값 판단
- 노드 C는 예:
  - 조도가 너무 낮으면 LED 켜기
  - 온도가 너무 높으면 팬/부저 켜기
- 임계값은 고정값 또는 사용자 설정값 사용

기능 4. 액추에이터 제어
- 노드 C는 노드 B로 제어 명령 전송
- 노드 B는 LED, 부저 등을 제어하고 결과 응답

기능 5. 모니터링 표시
- 노드 C는 최신 센서 상태, 제어 상태, 네트워크 연결 상태 표시
- UART 시리얼 출력만으로도 충분
- 가능하면 OLED에 간단 UI 표시

기능 6. 사용자 명령
- 노드 C에서 버튼 입력 또는 시리얼 입력으로 수동 명령 가능
- 예: `AUTO`, `MANUAL`, `LED_ON`, `ALARM_OFF`

**6. 권장 상태값**
- 센서값:
  - `temperature`
  - `light`
  - `pot`
  - `button`
- 제어 상태:
  - `led`

- 시스템 상태:

**7. 통신 설계**
권장 방식:

이유:
- 초반 구현이 UDP보다 이해하기 쉬움
- 데이터 누락 관리가 쉬움
- 발표 설명이 단순함

권장 서버:

- 노드 A, B는 클라이언트

메시지 형식:

- JSON도 가능하지만 초반에는 문자열 프로토콜이 구현이 편함

예시:
```txt
TYPE:SENSOR;NODE:A;TEMP:24.8;LIGHT:310;POT:602;BUTTON:0
```

파싱 규칙:
- `;`로 항목 구분
- `:`로 key/value 구분
- 한 줄 단위 메시지 처리

**안 2. Wi-Fi 미니 보안 시스템**

**1. 프로젝트 개요**
이 프로젝트는 3개의 Pico W가 `감지 노드`, `경보 노드`, `관리자 서버 노드` 역할을 맡아, 침입 또는 이상 상태를 감지하고 네트워크로 즉시 전파하여 경보를 울리며, 관리자가 이를 확인하고 해제하는 분산형 보안 시스템이다.

핵심 시나리오:
- 감지 노드가 버튼, 문열림 스위치, 움직임 감지 상태를 감시
- 이상 이벤트 발생 시 서버에 즉시 알림
- 서버는 경보 노드에 경고 명령 전송
- 경보 노드는 LED/부저 작동
- 관리자 노드는 경보 상태를 표시하고 해제 명령 가능

이 프로젝트는 `이벤트 기반 처리`, `실시간 네트워크 통신`, `상태 머신 설계`를 강조하기 좋다.

**2. 프로젝트 목표**
- 인터럽트 또는 폴링 기반 이벤트 감지 구현
- Wi-Fi 소켓을 이용한 이벤트 즉시 전달
- 상태 머신 기반 경보 시스템 설계
- 다중 노드 간 역할 분리 및 동기화
- 간단한 원격 제어 경험

**3. 시스템 구성**
노드 구성:
- `노드 A`: 감지 노드
- `노드 B`: 경보 노드
- `노드 C`: 관리자 서버 노드

구조:
```txt
[노드 A: 감지] ----Wi-Fi----\
                              > [노드 C: 관리자/서버] ----Wi-Fi---- [노드 B: 경보]
```

**4. 노드별 역할**
`노드 A: 감지 노드`
- 도어 센서 역할 버튼/스위치 입력
- PIR 센서가 있으면 움직임 감지
- 이상 이벤트 발생 시 즉시 서버 전송
- 평상시 heartbeat 메시지 송신

`노드 B: 경보 노드`
- 서버로부터 경보 명령 수신
- LED 점멸, 부저 경고음 출력
- 현재 경보 상태를 서버에 보고

`노드 C: 관리자 서버 노드`
- 감지 이벤트 수신
- 경보 상태 관리
- 경보 활성/해제 명령 전송
- 현재 침입 상태, 활성 경보, 최근 이벤트 로그 출력
- 관리자 승인 또는 해제 버튼 처리

**5. 핵심 기능 명세**
기능 1. 감지 기능
- 문 열림 스위치 또는 버튼 입력 감지
- 선택적으로 움직임 감지
- 입력 변화 시 즉시 이벤트 발생

기능 2. 즉시 이벤트 전송
- 감지 노드는 이벤트가 발생하면 서버에 즉시 전송
- 예:
  - `DOOR_OPEN`
  - `MOTION_DETECTED`
  - `PANIC_BUTTON`

기능 3. 경보 작동
- 서버는 정책에 따라 경보 노드에 명령 전송
- 경보 노드는 LED 점멸, 부저 반복 울림 수행

기능 4. 관리자 확인
- 서버 노드는 현재 상태 표시
- 사용자가 `ACK`, `RESET`, `DISARM` 명령 수행 가능

기능 5. 상태 머신 운영
시스템 모드 예시:
- `IDLE`
- `ARMED`
- `TRIGGERED`
- `ALARMING`
- `ACKNOWLEDGED`
- `RESET`

**6. 추천 상태 머신**
예시 흐름:
```txt
IDLE -> ARMED -> TRIGGERED -> ALARMING -> ACKNOWLEDGED -> RESET -> ARMED
```

설명:
- `IDLE`: 비활성 상태
- `ARMED`: 감시 활성 상태
- `TRIGGERED`: 이벤트 감지됨
- `ALARMING`: 경보 출력 중
- `ACKNOWLEDGED`: 관리자가 확인함
- `RESET`: 초기화 후 다시 감시 준비

**7. 통신 설계**
권장 방식:
- `TCP 소켓`
이유:
- 이벤트 누락 방지에 유리
- 상태 메시지와 명령 응답이 명확함

권장 서버:
- 노드 C를 서버
- 노드 A와 B는 클라이언트

메시지 형식 예시:
