# 스마트 환경제어 하우스 프로젝트 문서

## 1. 프로젝트 개요

본 프로젝트는 `Raspberry Pi Pico 2 W (RP2350)` 3대를 활용하여 집 내부 환경을 실시간으로 모니터링하고, 조명 및 환기 장치를 자동으로 제어하는 `스마트 환경제어 하우스`를 구현하는 것을 목표로 한다.

각 노드는 `환경 감지`, `장치 제어`, `중앙 관리` 역할로 분리되며, `Wi-Fi` 기반 통신을 통해 상호 연동된다.  
또한 각 팀원이 자신의 보드에서 독립적으로 기능을 검증한 뒤, 마지막에 통합하여 전체 시스템을 완성하는 방식으로 개발한다.

---

## 2. 과제 목표

- `조도`, `온도`, `습도` 센서를 이용하여 집 내부 환경을 실시간으로 측정한다.
- 측정된 환경 데이터에 따라 `LED 조명`과 `서보모터 기반 환기창`을 자동으로 제어한다.
- `AUTO` 모드와 `MANUAL` 모드를 구분하여 자동 제어와 수동 제어를 모두 지원한다.
- `Wi-Fi` 통신을 활용하여 각 노드가 독립적으로 동작하면서도 하나의 홈 IoT 시스템처럼 연동되도록 구현한다.
- `UART` 또는 `LCD/OLED`를 이용하여 현재 상태를 시각적으로 확인할 수 있도록 한다.

---

## 3. 시행 목적

본 프로젝트의 목적은 IoT 기술을 활용하여 주거 공간 내부의 환경을 자동으로 감지하고 제어할 수 있는 `스마트 환경제어 하우스 시스템`을 구축하는 것이다.

집 모형 내부를 하나의 실제 생활 공간으로 가정하고, `조도`, `온도`, `습도` 데이터를 실시간으로 측정하여 실내 환경 상태를 파악하고, 이를 기반으로 조명과 환기 장치를 자동으로 제어한다.

이를 통해 센서 입력, 마이크로컨트롤러 기반 제어, 액추에이터 구동, Wi-Fi 통신을 하나의 시스템으로 통합하는 과정을 직접 구현하며, 홈 IoT 시스템의 전반적인 동작 원리를 학습하는 데 목적이 있다. 또한 각 기능을 노드별로 분리하여 구현한 뒤 최종적으로 통합함으로써, 분산형 임베디드 시스템 설계와 협업 개발 방식에 대한 경험을 쌓고, 향후 웹 모니터링, 모바일 제어, 데이터 로깅 등으로 확장 가능한 구조를 설계하는 것을 목표로 한다.

## 7. 핵심 기능 명세

### 7-1. 환경 감지 기능

- 조도, 온도, 습도를 일정 주기로 측정한다.
- 측정값을 `UART`로 출력하여 각 노드에서 독립적으로 확인 가능해야 한다.
- 측정값은 중앙 관리 노드로 전송된다.
- 센서값 읽기 실패 시 오류 상태를 출력한다.

권장 주기:

- 조도: `0.5초 ~ 1초`
- 온습도: `1초 ~ 2초`
- 데이터 전송: `1초`

### 7-2. 자동 조명 제어 기능

- `AUTO` 모드에서만 동작한다.
- 조도값이 기준값보다 낮으면 조명을 켠다.
- 조도값이 기준값보다 높으면 조명을 끈다.
- 상태가 바뀔 때만 제어 명령을 전송한다.

권장 기준 예시:

- `light < 280` -> `LIGHT_ON`
- `light > 320` -> `LIGHT_OFF`

### 7-3. 자동 환기 제어 기능

- `AUTO` 모드에서만 동작한다.
- 온도 또는 습도가 기준값 이상이면 환기창을 연다.
- 온도와 습도가 정상 범위로 복귀하면 환기창을 닫는다.
- 서보모터를 사용하여 창문 개폐를 표현한다.

권장 기준 예시:

- `temp > 28` -> `WINDOW_OPEN`
- `humidity > 70` -> `WINDOW_OPEN`
- `temp <= 27` 그리고 `humidity <= 65` -> `WINDOW_CLOSE`

### 7-4. 수동 제어 기능

- 사용자가 `AUTO`와 `MANUAL` 모드를 전환할 수 있다.
- `MANUAL` 모드에서는 사용자가 조명과 창문을 직접 제어한다.
- `AUTO` 모드로 복귀하면 센서 기반 자동 제어를 재개한다.

지원 명령 예시:

- `MODE_AUTO`
- `MODE_MANUAL`
- `LIGHT_ON`
- `LIGHT_OFF`
- `WINDOW_OPEN`
- `WINDOW_CLOSE`

### 7-5. 상태 표시 기능

- 현재 모드, 조도, 온도, 습도, 조명 상태, 창문 상태를 표시한다.
- 기본 출력은 `UART`를 사용한다.
- 여유가 있으면 `LCD/OLED`에 상태를 출력한다.

출력 예시:

```txt
MODE: AUTO
LIGHT: 275
TEMP: 29.1
HUMI: 72
LAMP: ON
WINDOW: OPEN
WIFI: OK
```

### 7-6. 통신 기능

- 센서 노드는 환경 데이터를 전송한다.
- 중앙 관리 노드는 데이터를 바탕으로 제어 명령을 생성한다.
- 장치 제어 노드는 명령을 수신하여 실제 하드웨어를 동작시킨다.
- 각 노드는 연결 확인을 위한 상태 출력 또는 heartbeat를 가질 수 있다.

권장 MQTT 토픽 예시:

- `house/env`
- `house/cmd/light`
- `house/cmd/window`
- `house/status/actuator`
- `house/mode`

메시지 예시:

```txt
house/env
light=280,temp=27.3,humi=68

house/cmd/light
ON

house/cmd/window
OPEN

house/mode
AUTO
```

## 9. 노드별 세부 담당 내용

### 노드 A: 환경 센서 노드

- 조도센서 값 측정
- 온습도센서 값 측정
- 측정값 시리얼 출력
- 측정값을 정해진 형식으로 전송
- 필요 시 센서값 평균 처리 또는 임계값 비교용 1차 가공

추천 담당자:

- 센서 연결과 값 읽기 구현에 익숙한 사람
- 비교적 안정적으로 먼저 완성하고 싶은 사람

### 노드 B: 장치 제어 노드

- LED ON/OFF 제어
- 서보모터 각도 제어
- 명령 수신 시 장치 동작 수행
- 현재 장치 상태 시리얼 출력
- 필요 시 테스트 버튼 입력으로 로컬 검증 모드 구현

추천 담당자:

- 하드웨어 제어, PWM, 액추에이터 구현에 관심 있는 사람

### 노드 C: 중앙 관리 노드

- 센서값 수신 및 저장
- 조도/온습도 기준 비교
- AUTO/MANUAL 모드 관리
- 조명/창문 제어 명령 생성
- 현재 상태 출력
- 필요 시 모드 변경 버튼 또는 시리얼 명령 처리

추천 담당자:

- 전체 로직 정리와 통신 흐름을 맡을 사람
- 가장 복잡한 파트를 담당할 사람

## 11. 개발 원칙

- 각 노드는 개별 보드에서 독립적으로 실행 및 검증 가능해야 한다.
- 최종 통합 전까지는 각자 로컬 테스트를 완료한다.
- 마지막 단계에서 MQTT 토픽과 메시지 규격을 맞춰 통합한다.
- 발표 시에는 `센서 입력 -> 중앙 판단 -> 장치 제어` 흐름이 보이도록 구성한다.

## 12. 개발 완료 기준

- 노드 A가 환경 데이터를 정상 측정하고 출력한다.
- 노드 B가 조명 및 창문 제어를 정상 수행한다.
- 노드 C가 상태를 수신하고 제어 명령을 생성한다.
- `AUTO` 모드에서 조명과 환기가 자동 제어된다.
- `MANUAL` 모드에서 사용자가 직접 제어할 수 있다.
- 집 모형 기준으로 전체 시연이 가능하다.

## 14. 다른 대화창에 붙여넣기용 요약

우리 프로젝트는 `Pico 2 W` 3대를 이용한 `스마트 환경제어 하우스`이다.  
노드 A는 `조도센서`와 `온습도센서`로 환경을 측정하고, 노드 B는 `LED`와 `서보모터`로 조명 및 창문을 제어하며, 노드 C는 전체 상태를 관리하고 `AUTO/MANUAL` 모드를 제어한다.  
통신은 `Wi-Fi + MQTT` 기반으로 구성하며, 각 노드는 먼저 독립적으로 검증한 뒤 최종 통합한다.
